方案 A：从现有 `Action for` 延伸的批量分配

模型列表、Tab、搜索、角色徽标和原来的“Set as <role>”均保留。只有在主动选择批量分配时，才进入下面两步。

1
选择批量入口
模型已选中，原菜单仍可做单角色操作
Action for: gpt-5.3-codex
›Assign to roles...
Set as Default
Set as Advisor
Set as Developer
Set as Reviewer
Enter: continue   Esc: cancel

原有单角色项位置不变；批量入口固定在第一项。

2
多选目标角色
不隐藏当前配置，选择范围始终可见
Assign gpt-5.3-codex to roles
3 roles selected
› Filter roles...
[x]Developergpt-5.2
[x]Reviewerclaude-opus-4.6
[x]Test Engineergpt-5.2
[ ]Advisorgpt-5.2
[ ]Researchergemini-3-pro
Space: select   ↑↓: move   Enter: review   Esc: back
3
确认并反馈
先展示覆盖面，再提交所有变更
Confirm role assignments
Model: gpt-5.3-codex

3 role settings will be updated:

Developer: gpt-5.2 → gpt-5.3-codex
Reviewer: claude-opus-4.6 → gpt-5.3-codex
Test Engineer: gpt-5.2 → gpt-5.3-codex
› Apply 3 assignments
Enter: apply   Esc: back
Assigned gpt-5.3-codex to 3 roles.
排版模型名、角色名、当前值各占固定列，窄屏自动截断，避免徽标和长模型名把菜单撑乱。
安全多选不立即写配置；确认屏精确列出每一个覆盖项，Esc 可原路返回而不产生变更。
部署只扩展现有 `ModelSelectorComponent` 与 Settings 写入路径，不引入 Web 服务、额外依赖或新的配置文件。
