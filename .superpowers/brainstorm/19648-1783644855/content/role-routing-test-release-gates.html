测试矩阵、迁移检查与发布验收

发布不是“测试跑过”就结束，而是必须证明批量配置、任务选角、模型精确绑定和 Spawn 参数在真实链路中闭环。

1. 单元测试

  • 候选角色评分、阈值与 Top-2 分差
  • Role Contract 能力和权限校验
  • 配置选择器 canonicalize
  • provider/model/thinking 精确比较
  • 禁止 fallback、override 和 session 继承

2. 组件与集成测试

  • `Assign to roles...` 多选与返回
  • 一次确认写入多个 modelRoles
  • 失败不产生部分写入
  • 重新打开 `/model` 徽标一致
  • Settings → resolver → evidence 数据一致

3. Spawn 契约测试

  • 模拟真实 TaskTool Spawn 参数
  • 断言实际模型等于角色绑定
  • 错配注入必须在 Spawn 前阻止
  • 证据写入失败不得执行
  • 成功证据 exact_match 必须为 true
