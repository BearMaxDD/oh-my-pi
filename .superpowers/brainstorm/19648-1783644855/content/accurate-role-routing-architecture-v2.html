任务 → 正确角色 → `/model` 指定模型

模型分配以 `/model` 写入的角色配置为唯一事实源。角色选定后不再计算“推荐模型”，也不允许使用其他角色、Agent Override 或当前会话模型替代。

唯一模型来源用户通过 `/model` 为角色选择 provider、model 和 thinking levelsettings.modelRoles[validated_role_id]
01 TASK

归一化任务

提取目标、文件范围、验收条件、运行面与显式角色。

task_evidence
02 ROLE

确定正确角色

规则候选、高置信度直达；歧义任务由 Advisor 仲裁。

candidate_roles[]
03 CONTRACT

角色契约校验

能力、读写权限、阶段与 Advisor 要求全部匹配。

validated_role_id
04 BINDING

读取 `/model` 绑定

只调用 getModelRole(validatedRoleId)，缺失立即阻止。

configured_selector
05 EXACT RESOLVE

精确解析模型

解析 provider/model/thinking；不可用、歧义或通配命中均不执行。

canonical_model_id
06 ASSERT

Spawn 前一致性断言

配置选择器、解析模型和实际请求参数必须完全一致。

exact_match = true
07 RUN

执行并记录证据

携带已冻结角色与模型启动子代理，记录全链路来源。

routing_evidence
强不变量：actual_model.provider/model/thinking === canonicalize(settings.modelRoles[validated_role_id])。任何不相等都返回错误，不进入 TaskTool Spawn。

允许的结果

configured_exact角色绑定存在，精确模型已注册且认证可用，正常执行。
explicit_role_exact用户显式指定角色，契约通过后仍只执行该角色在 `/model` 中绑定的模型。
bulk_assigned_exact批量分配写入多个角色后，每个角色各自读取同一个明确选择器。

必须阻止的结果

role_model_unconfigured提示先用 `/model` 为该角色分配模型。
role_model_unavailable绑定模型不存在、未认证或不可用，不寻找替代模型。
role_model_mismatchSpawn 即将使用的模型与配置不一致，立即中止并输出两边值。
移除 role-bound fallbackModelRoles忽略 agentModelOverrides禁止继承 active session model禁止通配符命中其他模型禁止静默 default

代码落点：Settings.getModelRole 读取唯一绑定；resolveTaskModelRouting 增加 strict role-bound 分支；TaskTool.#runSpawn 执行一致性断言；createModelRoutingEvidence 记录 configured / resolved / actual 三元组。
